# Как настраивал тесты для GitHub API:

Этот документ — не просто инструкция. Это **конспект моих шагов, проблем и решений** при настройке тестирования GitHub API. Здесь есть детали, которые понимаешь только на практике.

## 1. Как получал токен (и как не словить 403 ошибку)
Первая же проблема: в разделе Developer settings GitHub предлагает **два типа токенов**: "Fine-grained" и "Classic".
*   **Fine-grained** — новый, с детальными правами, но сложнее.
*   **Classic** — старый, но проще для обучения.

**Я выбрал Classic**, потому что для чтения профиля и репозиториев хватило двух галок:
-   `read:user` (иначе `GET /user` вернёт 403)
-   `public_repo` (иначе не увижу свои репозитории)

## 2. Настройка Postman: как не перепутать /user и /users
Самая частая моя ошибка в начале:
-   `GET /user` — возвращает **мой** профиль (требует токен).
-   `GET /users/octocat` — возвращает профиль **любого публичного пользователя** (токен не нужен).
