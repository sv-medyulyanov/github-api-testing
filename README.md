# GitHub API Testing Project

Практический проект по тестированию REST API GitHub с аутентификацией через **Personal Access Token (PAT)**.

## Цель проекта
- Освоить **аутентификацию в REST API** с использованием **Bearer Token**.
- Протестировать ключевые эндпоинты GitHub API.
- Настроить **автоматические проверки (тесты)** в Postman.
- Создать портфолио с реальными, воспроизводимыми примерами.
- Применить на практике **принцип минимальных привилегий** при выдаче прав токену.

## Что протестировано
1.  **Аутентификация** — настройка безопасной работы с токеном через **переменные окружения Postman**.
2.  **Профиль пользователя** — эндпоинт `GET /user` (контекстный запрос с авторизацией).
3.  **Репозитории пользователя** — эндпоинты:
    - `GET /user/repos` (список своих репозиториев).
    - `GET /repos/{owner}/{repo}` (информация о конкретном публичном репозитории).
    - `GET /repos/{owner}/{repo}/issues` (обработка сценария с пустым списком данных).

## Ключевые навыки в проекте
- Работа с **REST API** (GET-запросы, анализ структур JSON-ответов).
- **Безопасность:** Использование переменных окружения для хранения секретов, работа с токенами с ограниченной областью видимости (`scopes`).
- **Автоматизация тестов:** Написание проверок на **JavaScript** в Postman (статус-коды, наличие полей, структура данных).
- **Документирование:** Создание структурированных тест-кейсов и отчётов в **Markdown**.
